Connecting the GPIO Breakout

For most (all) of these sensor tutorials, we'll be using the GPIO breakout connected to the Raspberry Pi. So let's see how that set up works and learn a little about what's going on. The Raspberry Pi has 40 GPIO pins. GPIO stands for "general purpose input and output". This allows us to get information (and power) into and out of our Raspberry Pi.

These pins are to communicate with other circuitry such as extension boards, custom circuits and much more. You can make some pretty cool stuff by using these pins. 

There is a caveat here: Experimenting with the GPIO is risky and has a chance of bricking your Raspberry Pi (and we don't want that!). Knowing the power limitations of the GPIO pins and also understanding what you’re plugging in is a must. So please read this section first before going into the other exercises. And as always, ask plenty of questions!

Below is a diagram of the GPIO pins on the evolutions of the Raspberry Pis. We have the newest ones, so the pin diagram on the right is what we're looking at.

Here are some details of the labeling you see in the diagram:

  • GPIO are your standard pins that simply be used to turn devices on and off. For example, an LED.
  • I2C (Inter-Integrated Circuit) pins allow you to connect and talk to hardware modules that support this protocol (I2C Protocol). This will typically take up 2 pins.
  • SPI (Serial Peripheral Interface Bus) pins can be used to connect and talk to SPI devices. Pretty much the same as I2C but makes use of a different protocol.
  • UART (Universal asynchronous receiver/transmitter) are the serial pins used to communicate with other devices.
  • DNC stands for do not connect, this is pretty self-explanatory.
  • The power pins (+) pull power directly from the Raspberry Pi.
  • GND are the pins you use to ground your devices. It doesn’t matter which pin you use as they are all connected to the same line.

I've already set up all of our Raspberry Pis so that we can use the pins (there are some configuration steps detailed here, if you're curious).

Because it's a physical connection, though, you'll need to set up the breakout board yourself. To do this, you'll need:

  • A Raspberry Pi 
  • A GPIO breakout kit (it should look like this; you need both pieces)
  • A breadboard (it should look like this; for those who are interested, I found this quick description of the basics of a breadboard really interesting)

A breakout kit allows you take all the pins via a ribbon cable and connect them to a breadboard or a difference device. This is a lot easier and safer than working in and around the Raspberry Pi.

Take the "lid" off the Raspberry Pi. I also recommend unplugging it for this step, just so you have a good range of motion. When connecting the ribbon cable, you need to make sure it is connected so it is facing away from the board. Look at the picture below to see what the proper set up should be.

Go ahead and make these connections. Be gentle but firm. You will likely have to use a little strength to get everything seated correctly. Make sure the T of the breakout straddles the midline of the breadboard and be sure to press it all the way in. It's not important which column of holes it fits into, though.

Now take a minute to take a closer look at the T. You'll see that the "pins" are named to match the pin diagram above. Specifically, note the 3V3 pin (that's where we can get power from the RPi) and find at least one GND (ground) pin. We'll use those in every sensor setup. Then there's pins with labels like "#4" and "#5", etc. These are places we can connect input signals to our Raspberry Pi. It's where our sensors will send information back into the RPi.

Temperature Sensor

We're going to connect a DS18B20 waterproof sensor to our Raspberry Pi. This probe can provide temperatures over a one wire interface. Even better, the sensor is waterproof making it work well in a lot of environments. The sensor is pretty accurate (within 0.05°C of the correct temperature) and has a huge sensing range (up to 125°C/260°F). But it is recommended to keep it under 100°C. The sensor is a digital device (it has an on-board analog to digital converter), which is what makes it so easy for us to hook it up to the Raspberry Pi.

First, though, let's make sure you have all of the following things handy:

  • A Raspberry Pi 
  • A GPIO breakout kit (it should look like this; you need both pieces)
  • A breadboard (it should look like this)
  • Some breadboard wire (you should need roughly five pieces)
  • A DS18B20 temperature sensor (it should look like this)
  • A 4.7 kohm resister (it should look like this; make sure the colored bands match! If you want to learn a little more about what the color bands on a resister mean, check out this link)

The circuit that we will need to build is pretty straight forward as we only need a resistor and the temperature sensor. But if this is your first time working with wiring on a breadboard, it can seem a little daunting. We'll get the hang of it.

Make sure you have the GPIO breakout assembled as depicted in the previous section.

  1. First connect the 3v3 pin in the breakout's T (it's on the top left, if you're looking at it straight on) up to the positive rail of the breadboard (that's the column all the way to the left (or right) with a red + sign).
  2. Connect a ground pin from the breakout (one of the pins labeled GND) to the ground rail on the breadboard (that's the second leftmost/rightmost column on the breadboard with a blue - sign).
  3. Now place the DS18B20 sensor onto the breadboard. Put it somewhere down below the T so that the red, yellow, and black wires are all in the same column of the breadboard but each is in a different row. I found it easiest to separate them by two rows each, just to give myself some room to work.
  4. Place a 4.7 kohm resistor (double check, and make sure you have the right resistor! Remember, it should look like this) between the positive lead and the output lead of the sensor. More specifically, put one end of the resister (it's ok to bend the ends... you'll have to) in the same row as the red wire from the temperature sensor. Put the other end of the resister in the same row as the yellow wire from the temperature sensor
  5. Next, we need to provide power to the temperature sensor. We'll get that straight from the Raspberry Pi, through the positive rail of the breadboard. So place a wire from the positive lead of the temperature sensor (that's the red wire) to the positive rail of the breadboard (the one with the red + sign). (Again, what you're doing is connecting the row of the red wire from the temperature sensor with the column with the red + sign.)
  6. We also need to ground our temperature sensor. Place a wire from the ground lead on the temperature sensor (that's the black wire) to the ground rail on the breadboard (the column with the blue - sign).
  7. Finally, we'll want to actually get some data from our temperature sensor, so we need to send the signal from the output lead (that's the yellow wire) back to the Raspberry Pi. Let's send it in through pin #4 (you can choose whichever you want). So place a wire from the row with the temperature sensor's yellow wire to the row where "#4" appears in the breakout T (you might need a slightly longer wire for this one).

Alright. That's the circuit. Assuming your GPIO is all solidly connected, you can plug your Raspberry Pi back in, let it boot up, and you'll be ready for the next step.

We're going to write a Python on the Pi script to get the data out of our temperature sensor, But before we do that, we need to setup the Pi so it can actually read data from the sensor. To do this we need to add OneWire support.

Go back to the VNC connection to your Pi. If it has closed, just relaunch it. You'll need to open a terminal to issue these commands coming up. The terminal is available in the taskbar at the top left (it's a black rectangle with a blue header and a white >_ inside the black).

Open up the boot config file by typing 

sudo nano /boot/config.txt

Sudo is a command that gives you read, write, and execute permissions to all of the files on the RPi. We'll use it all of the time so that we're basically always running in a privileged mode. Nano is the name of a very simple text editor. We'll use it for easy jobs like this one. At the bottom of this file (just use the down arrow to get to the bottom), enter:

dtoverlay=w1-gpio

Once done save and exit by pressing ctrl x and then y. Now reboot the Pi by running the following command.

sudo reboot

Once the Raspberry Pi has booted back up we need to run modprobe so we can load the correct modules. Type each of the following into the terminal, pressing enter after each command

sudo modprobe w1-gpio
sudo modprobe w1-therm

Now change into the devices directory (using the cd command) and use the ls command (ls is for "list") to see the folder/files in this directory.

cd /sys/bus/w1/devices
ls

Now run the following command, but change the number after cd to whatever directory was listed when you ran the ls command just above.

cd 28-000007602ffa

Now run the following command.

cat w1_slave

Cat is a command that simply prints the contents of a file to the screen. In this case, w1_slave is a file where the data from your temperature sensor is stored. So this command should output some data, but it's not particularly easy to read. The first line should have a YES or NO at the end of it. If it is YES, then a second line with the temperature should appear. At the end of this line will be some number like t=26250; we just need to do a little bit of math to make this a temperature we understand. For instance, to get a celsius value, just divide by 1000.

Now it’s time to move onto the python script. We can create this python script using the nano editor again, but the RPi comes with a python editor as well that will provide some "syntax highlighting" (i.e., color) for our python scripts. So let's use that. Click on the little raspberry icon at the top left of the RPi desktop. Under "Programming", select "Python 3". This will open the python shell. Next, click "File" then "New File". This will open a blank screen for editing. Let's go ahead and name/save our file (even though it's empty). In this new blank window, select "File", then "Save As". Type "thermometer.py", then press Save. This creates a new file (called thermometer.py). The ".py" extension on this file indicates that it is a python script. Python is just another programming language.

OK. Now we can start writing our code.

First, type the following three lines at the top of the file

import os
import glob
import time

These are import statements that get us access to some packages we need. The first one allows us to make some calls into the operating system (os). The glob package is going to allow us to match file names that match a certain pattern (so we don't have to know the entire file name, for instance the name of the file where the sensor data is dumped... above it started with 28, but could have had all kinds of things following the 28). Finally, the time package allows us to do things like "sleep" in our program to allow the Pi to do other things.

Next type:

os.system('modprobe w1-gpio')
os.system('modprobe w1-therm')

These should look familiar; they're the exact commands we typed in the terminal above to check to see if our sensor was working. They're doing the same thing here (starting up the interface that will read from the sensor), just programmatically.

Next type:

base_dir = '/sys/bus/w1/devices/'
device_folder = glob.glob(base_dir + '28*')[0]
device_file = device_folder + '/w1_slave'

These three lines are finding that file from which the sensor data can be read. Again, they should look kind of familiar; it's basically the same thing we did above to check if the sensor was working. The first line specifies the devices directory; the second line uses the glob package to select any subfolder that starts with 28. Finally, the last line gets the specific file, called w1_slave.

Next we're going to define a function that will just read the two lines from the w1_slave file (the two we printed out before... remember the first one ended in "YES" and the second one ended in the temperature value). Type:

def read_temp_raw():
  f = open(device_file, 'r')
  lines = f.readlines()
  f.close()
  return lines

This code may look a little confusing. The first line defines (hence "def") a function called read_temp_raw. The first thing it does is open the device file using the file name we already defined. Then it just saves the lines (there will be two of them, but there's nothing about this code that tells you that) into the variable called lines. Then it closes the file and "returns" the lines variable to whoever called the function.

Next, we're going to define a second function. This function looks a bit more complicated. It's going to read the lines (we call them "strings" because they contain strings of textual material) and then do the math to convert the temperature values to celsius and Fahrenheit. It's going to effectively do the same thing you did. First, it will scan the first line to see if it contains "YES" at the end. If it does, then it will scan the second line until it finds "t=" and then it will read the number. Finally, it will do a little math.

def read_temp():
  lines = read_temp_raw()
  while lines[0].strip()[-3:] != 'YES':
    time.sleep(0.2)
    lines = read_temp_raw()
  equals_pos = lines[1].find('t=')
  if equals_pos != -1:
    temp_string = lines[1][equals_pos+2:]
    temp_c = float(temp_string) / 1000.0
    temp_f = temp_c * 9.0 / 5.0 + 32.0
    return temp_c, temp_f

The while loop (the second line of the function) just waits until the end of the first line of the file is YES. The if (the sixth line of the function) checks to make sure "t=" appears in the second line before it tries to use the value. When this function completes, it returns a pair of values made up of the celsius value and the Fahrenheit one.

Now we're going to write a program that will do this repeatedly until you tell it to stop. That is, the program will just repeatedly open the file, read the contents, convert them to celsius and Fahrenheit, write them to the screen, take a nap (for one second), then do it all again. So at the bottom of your file, underneath your function definitions, type the following:

while True:
  print(read_temp()) 
  time.sleep(1)

And that's it! We've written our program to get the sensor values off of the temperature sensor. And now we can use them in the RPi for something else! To save your work, select "File" then "Save" (or just type CTRL-S). In general, it's NEVER a bad idea to save your work periodically while you're typing. 

OK. Now let's run it. To get the satisfaction out of all of this working, make sure your circuit is all still connected and your GPIO breakout is all still seated right. In the terminal on the RPi, type 

sudo python thermometer.py

If it's all working right, you should see the temperature values written to your screen. Try blowing hot air onto the temperature probe. You should see the numbers go up. Good work!

Light Sensor

In this exercise, we're going to connect a photo resister (which can sense light levels) up to the GPIO pins on the Pi. We'll write a simple python script to gather and use the data from the light sensor. First, let's make sure you have all of the following things handy:

  • A Raspberry Pi 
  • A GPIO breakout kit (it should look like this; you need both pieces)
  • A breadboard (it should look like this)
  • Some breadboard wire (you should need roughly four pieces)
  • A light sensor (it should look like one of these guys)
  • A 1uF capacitor (it should look like this; for more information on what a capacitor is and does, check this out)

The light dependent resistor (LDR) sensor (also known as a photoresistor) is the most important piece of equipment in our circuit (obviously). We will use it to detect whether the ambient surroundings are light or dark. In lighted conditions, this sensor will have a resistance of only a few hundred ohms, while in the dark it can have a resistance of several mega ohms.

The capacitor in our circuit is there so that we can measure the resistance of the LDR sensor. A capacitor essentially acts like a battery charging up while receiving power and then discharging when no longer receiving power. Using this in series with the LDR, we can work out how much resistance the LDR has and thus whether it is light or dark.

The circuit itself is pretty easy to connect up.

  1. First connect the 3v3 pin in the breakout's T (it's on the top left, if you're looking at it straight on) up to the positive rail of the breadboard (that's the column all the way to the left (or right) with a red + sign).
  2. Connect a ground pin from the breakout (one of the pins labeled GND) to the ground rail on the breadboard (that's the second leftmost/rightmost column on the breadboard with a blue - sign).
  3. Place the LDR sensor onto the breadboard so that the two legs are in two different rows (I connected mine across the "ravine" in the breadboard since the rows on either side of the ravine are independent).
  4. Connect one side of the LDR to the positive rail (that is, in the same row as one leg of the LDR, put one end of a breadboard wire; connect the other end of this breadboard wire to the rail where you connected the power from the Pi).
  5. Connect the other side of the LDR to #4 on your breakout board. This is where the measurement of the resistance across the LDR will be shuttled back to the Pi. (Specifically, take a piece of breadboard wire and put one end in the same row as the second leg of the LDR. Put the other end of the wire in the same row as "#4" on the breakout T.)
  6. Finally, pick up the capacitor. It has two legs; one is longer than the other. The shorter leg is the negative pin of the capacitor (you can also check this; where the shorter leg hits the capacitor itself, you should see a silver strip on the capacitor, again indicating that this is the negative pin). Put the negative pin of the capacitor in the negative (ground) rail of your breadboard. Put the positive pin of the capacitor in the same row of the breadboard as the second leg of the LDR (the one connected to #4 on your Pi). 

The code for this project is pretty simple and will tell us roughly whether it is light, shady or completely dark. However, it's a little rough because of the difference between analog and digital sensors. The only significant challenge is that the Pi doesn't have analog pins (it only has digital ones). If you want to dig into the details of the difference, check out this page. The quick difference is that you can think about analog waves as smooth and continuous and digital ones are discrete (e.g., just take values of 1s and 0s). This is why the (digital) temperature information is so easy to read in code (see the temperature exercise), while this light sensor is a bit more difficult because we can't accurately measure the variance in resistance in our input (it's an analog signal). Instead we will be measuring the time it takes for the capacitor to charge and send the pin high. This is an easy but somewhat inaccurate way of telling whether it is light or dark.

So now we're ready to move onto the python script.

We can create this python script using the nano editor again, but the RPi comes with a python editor as well that will provide some "syntax highlighting" (i.e., color) for our python scripts. So let's use that. Click on the little raspberry icon at the top left of the RPi desktop. Under "Programming", select "Python 3". This will open the python shell. Next, click "File" then "New File". This will open a blank screen for editing. Let's go ahead and name/save our file (even though it's empty). In this new blank window, select "File", then "Save As". Type "light_sensor.py", then press Save. This creates a new file (called light_sensor.py). The ".py" extension on this file indicates that it is a python script. Python is just another programming language.

We're now going to walk through creating the entire contents of this file.

First, type the following two import statements at the top of the file:

import RPi.GPIO as GPIO
import time

These statements tell the program to import two packages: one is to interact with the GPIO capabilities (and we'll refer to that package as GPIO throughout the program) and the second is the time package, which allows us to do things like sleep for small periods of time to wait on sensor data.

Now add these three lines:

GPIO.setmode(GPIO.BOARD)
#define the pin that goes to the circuit
pin_to_circuit = 7

The first statement sets the GPIO mode to GPIO.BOARD, which means that all of the numbering that we will use in this script will refer to the physical numbering of the pins (and not necessarily the easier numbering scheme printed on the GPIO breakout). If you need a reminder of this, scroll back up and take a look at that pin layout figure. The second line is a comment (comments are handy sometimes to remind us what the purpose of things are!).  Since we only have one input/output pin we only need to set one variable. Set this variable to the number of the pin you have acting as the input/output pin; if you followed the instructions above, it will be pin number 7 (this is the physical pin number of the GPIO #4 pin). This is what the third line does.

Next we're going to add a function called rc_time that requires one parameter: the pin number to the circuit. In this function we initialize a variable called count; we will return this value once the pin goes to high.

def rc_time (pin_to_circuit):
  count = 0
 
  #Output on the pin for 
  GPIO.setup(pin_to_circuit, GPIO.OUT)
  GPIO.output(pin_to_circuit, GPIO.LOW)
  time.sleep(0.1)

  #Change the pin back to input
  GPIO.setup(pin_to_circuit, GPIO.IN)
 
  #Count until the pin goes high
  while (GPIO.input(pin_to_circuit) == GPIO.LOW):
    count += 1

  return count

Here's what this function is doing. We set our pin to act as an output and then set it to low, we then have the script sleep for 10ms. After this, we then set the pin to become an input and then we enter a while loop. We stay in this loop until the pin goes to high, this is when the capacitor charges to about 3/4. While we're waiting on the pin to go to high, we increment the counter every time the loop executes. Once the pin goes high we return the count value to the main function. The darker it is, the longer it will take the capacitor to charge, so the higher the returned value will be. When it is lighter, the capacitor will charge faster, so the returned value will be lower.

Now that we have this function defined, all we have to do is write the main behavior of the program, which will just repeatedly call the the rc_time function until interrupted by a keyboard input (CTRL-C) from the user. To accomplish this, add the following lines of code:

#Catch when script is interrupted, cleanup correctly
try:
  # Main loop
  while True:
    print rc_time(pin_to_circuit)
except KeyboardInterrupt:
  pass
finally:
  GPIO.cleanup()

As complicated as it looks, this is actually surprisingly simple code. The try block basically instructs the python interpreter (that's executing the code) to continue to do the stuff inside the while loop ("while True" means "do this forever") with one exception: a "KeyboardInterrupt". If that interrupt is received, we just quit, but before we quit in this state, we want to leave the GPIO in a clean state, so we always (finally) call GPIO.cleanup().

To save your work, select "File" then "Save" (or just type CTRL-S). In general, it's NEVER a bad idea to save your work periodically while you're typing. 

Back in the command line of the terminal, type:

sudo python light_sensor.py

to see your code in action. Check whether its working by simply shading (and then unshading) the light sensor with your hand. Great work!

Motion Sensor

In this exercise we’re going to build a Raspberry Pi motion sensor that makes use of a PIR (Passive Infrared Sensor) sensor. First up, here's the stuff we're going to need:

  • A Raspberry Pi 
  • A GPIO breakout kit (it should look like this; you need both pieces)
  • A breadboard (it should look like this)
  • Some breadboard wire (you should need roughly five pieces)
  • A PIR sensor (it should look like this)
  • A 100 ohm resister (it should look like this; make sure the colored bands match! If you want to learn a little more about what the color bands on a resister mean, check out this link)

A PIR sensor is most commonly seen in security systems to detect movement before setting off an alarm. They detect motion whenever there is a change of infrared temperature in the field of view. Most PIR sensors have some adjustable screws on them that will allow you to adjust both their timing and sensitivity. The time will allow you to set a delay before the sensor will go off (i.e., before it sends a high signal), which is usually in the range of 2 to seconds. The sensitivity is how much movement needs to occur before the sensor will go off.

The circuit itself is pretty easy to connect up.

  1. First connect 5V pin in the breakout's T (it's on the top right, if you're looking at it straight on) up to the positive rail of the breadboard (that's the column all the way to the left (or right) with a red + sign). [Note: this is different power than used in the other two exercises.]
  2. Connect a ground pin from the breakout (one of the pins labeled GND) to the ground rail on the breadboard (that's the second leftmost/rightmost column on the breadboard with a blue - sign).
  3. Insert the white connector into the PIR sensor. On the PIR board, you'll notice that "GND", "OUT", and "+5V" are all marked. Make sure that the black cable lines up with GND and the red cable lines up with +5V.
  4. Run a wire from physical pin 11 (it's labeled #17 on the GPIO breakout board) to the breadboard.
  5. Place one end of the 100-ohm resistor in the same row as the connection to pin 11. Place the other end of the resistor in another row.
  6. Connect the yellow wire of the PIR sensor to the second end of the resistor (i.e., in the same row as the second end of the resistor)
  7. Finally, connect the red wire of the PIR sensor to the 5V rail and the black wire of the PIR sensor to the GND rail.

In order to bring our Raspberry Pi Motion sensor circuit to life, we now need to do a little programming. Head on over to your VNC connection to the RPi. (Relaunch it if necessary.)

We can create this python script using the nano editor again, but the RPi comes with a python editor as well that will provide some "syntax highlighting" (i.e., color) for our python scripts. So let's use that. Click on the little raspberry icon at the top left of the RPi desktop. Under "Programming", select "Python 3". This will open the python shell. Next, click "File" then "New File". This will open a blank screen for editing. Let's go ahead and name/save our file (even though it's empty). In this new blank window, select "File", then "Save As". Type "motion_sensor.py", then press Save. This creates a new file (called motion_sensor.py). The ".py" extension on this file indicates that it is a python script. Python is just another programming language. 

For this exercise we will set our GPIO mode to reference the physical numbering of the pins rather than the numbering on the breakout board. This is a little easier to understand/remember as all the pins are numbered in order. We also have to setup our GPIO pin to be either an output or an input. Because we want to detect motion from the PIR sensor, we want to set it up as an input. 

Type the following lines:

import RPi.GPIO as GPIO
import time

pir_sensor = 11
GPIO.setmode(GPIO.BOARD)
GPIO.setup(pir_sensor, GPIO.IN)

current_state = 0
count_events = 0

The first two lines are import statements. These statements tell the program to import two packages: one is to interact with the GPIO capabilities (and we'll refer to that package as GPIO throughout the program) and the second is the time package, which allows us to do things like sleep for small periods of time to wait on sensor data.

The next line declares a variable to hold the physical pin number for the PIR input (if you hooked it up as suggested, 11 is the right number to use here). The next couple of lines set up the GPIO to tell it to use the physical pin numbering and that the PIR sensor should be an input sensor.

The second to last line in this set declares a variable called current_state, which is a global variable that will be used within our program to capture whether motion has been detected. The last line will count the number of detection events we've had in total during the program's run.

Now enter the following lines of code, which define the main portion of our program:

try:
  while True:
    time.sleep(0.1)
    current_state = GPIO.input(pir_sensor)
    if current_state == 1:
      count_events += 1
      print("Motion detected: %s" % (count_events))
      time.sleep(5)
except KeyboardInterrupt:
  pass
finally:
  GPIO.cleanup()

As complicated as it looks, this code is pretty straightforward (and this program doesn't use any functions). The try block basically instructs the python interpreter (that's executing the code) to continue to do the stuff inside the while loop ("while True" means "do this forever") with one exception: a "KeyboardInterrupt". If that interrupt is received, we just quit, but before we quit in this state, we want to leave the GPIO in a clean state, so we always (finally) call GPIO.cleanup().

Within the while loop is the body of our program. First, the program will sleep for 100 milliseconds. Then it will get the PIR sensor input. If the state of the input is 1, then motion has been detected. We increment the number of detection events we've had so far (count_events), then print out the text "Motion detected" and the current value of the counter. Then we sleep for five seconds and do it all again.

And that's all there is to the code. To save your work, select "File" then "Save" (or just type CTRL-S). In general, it's NEVER a bad idea to save your work periodically while you're typing. 

Back in the command line of the terminal, type:

sudo python motion_sensor.py

to see your code in action. Check whether its working by moving (and then not moving) in the vicinity of the motion sensor. Be careful, it has a very wide view. You can also play with the sensitivity screw as desired to change the behavior.
